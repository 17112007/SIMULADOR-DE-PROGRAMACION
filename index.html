Primer Trimestre – Fundamentos

¿Qué es un algoritmo? Un algoritmo es una secuencia finita de instrucciones claras y ordenadas que permite resolver un problema de manera eficiente. Su correcta planificación es crucial, ya que un algoritmo mal diseñado puede generar errores, retrasos o resultados incorrectos. Los algoritmos pueden representarse mediante pseudocódigo, diagramas de flujo o directamente en un lenguaje de programación, facilitando su comprensión y desarrollo. Además, son la base de cualquier software, desde aplicaciones simples hasta sistemas complejos de inteligencia artificial.
Características:
  • Orden secuencial de pasos.
  • Instrucciones claras y precisas.
  • Debe tener inicio y fin definidos.
  • Representable en pseudocódigo, diagramas o código.
  • Base para cualquier programa o sistema.
 Ejemplo: Una receta para preparar un café, indicando paso a paso cómo hervir agua, añadir café y servir. Pasos lógicos Los pasos lógicos son las acciones que deben ejecutarse en un orden coherente para garantizar el funcionamiento correcto de un algoritmo. Cada instrucción tiene sentido solo si se coloca en la secuencia adecuada; cambiar el orden puede provocar fallos o resultados inesperados. Comprender los pasos lógicos permite optimizar procesos y reducir la complejidad del código, siendo una habilidad esencial para todo programador.
Características:
  • Secuencia coherente de acciones.
  • Facilitan la comprensión del algoritmo.
  • Permiten optimizar y depurar procesos.
  • Base de la lógica computacional.
 Ejemplo: Servir el café antes de beberlo, siguiendo un orden lógico. Entrada y salida de datos La entrada de datos consiste en la información que recibe un programa para procesarla, mientras que la salida es el resultado que se devuelve después de procesarla. Esta interacción con el usuario o con otros sistemas permite que los programas sean dinámicos y útiles. Comprender bien este concepto es clave para diseñar aplicaciones que respondan correctamente a diferentes escenarios y para garantizar que los resultados sean confiables.
Características:
  • Entrada: información que recibe el programa.
  • Salida: resultados que el programa devuelve.
  • Permiten interacción con usuarios o sistemas.
  • Clave para la validación y confiabilidad.
 Ejemplo: Ingresar números (Entrada) -> Mostrar suma total (Salida). Variables Las variables son espacios en memoria que almacenan datos que pueden cambiar durante la ejecución de un programa. Permiten guardar información temporal, como resultados intermedios de cálculos, y manipular datos de forma flexible. Conocer su tipo, alcance y duración es fundamental para programar de manera eficiente y evitar errores como sobrescribir información importante. Su correcta utilización mejora la legibilidad y mantenibilidad del código.
Características:
  • Almacenan información temporal.
  • Pueden cambiar su valor durante la ejecución.
  • Tipos: enteros, decimales, texto, booleanos.
  • Importantes para cálculos y lógica del programa.
 Ejemplo: Un cronómetro que guarda el tiempo transcurrido y lo actualiza cada segundo. Tipos de datos Los tipos de datos determinan la clase de valor que puede almacenar una variable, como números enteros, decimales, texto o valores lógicos. Conocer los tipos de datos permite optimizar la memoria, garantizar cálculos correctos y evitar errores de compatibilidad. Cada tipo de dato tiene propiedades y operaciones específicas que se pueden realizar sobre él.
Características:
  • Definen la naturaleza del valor almacenado.
  • Permiten optimizar memoria y cálculos.
  • Facilitan interacción entre variables y funciones.
  • Importantes para control de errores.
 Ejemplo: Número (Int) vs Texto (String), como edad = 18 y nombre = "María". Enteros Los enteros son números sin parte decimal, incluyendo positivos, negativos y el cero. Son esenciales para conteos, índices de arreglos, ciclos de repetición y operaciones matemáticas básicas. Permiten realizar cálculos precisos y operaciones discretas, siendo la base para estructuras de control.
Características:
  • Números sin decimales.
  • Incluyen positivos, negativos y cero.
  • Se usan en conteos y ciclos.
  • Fundamentales para cálculos exactos.
 Ejemplo: 7, -2, 0, 15. Decimales Los números decimales contienen una parte fraccionaria y permiten representar valores más precisos que los enteros. Se utilizan en cálculos de precios, medidas, estadísticas y cualquier situación que requiera exactitud. Comprender su manejo y formato es fundamental para evitar errores de redondeo o pérdida de información.
Características:
  • Contienen parte fraccionaria.
  • Se usan para cálculos precisos.
  • Representados por float o double en programación.
  • Claves en medidas, estadísticas y finanzas.
 Ejemplo: 12.50, 3.1416. Cadenas de texto Las cadenas de texto son secuencias de caracteres que representan palabras, frases o símbolos, generalmente encerradas entre comillas. Se usan para almacenar nombres, mensajes, instrucciones y cualquier información textual. Permiten concatenar, modificar y comparar texto de manera eficiente dentro de un programa.
Características:
  • Secuencias de caracteres.
  • Encerradas entre comillas.
  • Se pueden concatenar y modificar.
  • Claves para interacción y salida de datos.
 Ejemplo: "Hola Mundo", "Unidad Educativa Dolores Sucre". Booleanos Los valores booleanos representan verdadero o falso y son fundamentales para el control de flujo en los programas. Se utilizan en condicionales, bucles y validaciones, determinando qué camino debe seguir el algoritmo. Permiten combinar condiciones complejas con operadores lógicos para lograr resultados precisos.
Características:
  • Solo tienen dos valores: True o False.
  • Clave para decisiones y validaciones.
  • Se combinan con operadores lógicos.
  • Usados en control de flujo y bucles.
 Ejemplo: True / False, usado en verificar si un usuario está registrado. Operadores matemáticos Los operadores matemáticos son símbolos que permiten realizar operaciones aritméticas como suma, resta, multiplicación y división. Son la base para cualquier cálculo en programación, desde operaciones simples hasta fórmulas complejas. Con su uso adecuado se pueden crear algoritmos precisos y eficientes.
Características:
  • Permiten realizar operaciones aritméticas.
  • Incluyen +, -, *, /.
  • Claves en cálculos y algoritmos.
  • Fundamentales para manipulación de datos numéricos.
 Ejemplo: 10 + 5 = 15, 20 * 2 = 40. Operadores lógicos Permiten combinar y comparar condiciones dentro de un programa para tomar decisiones complejas. Los principales son AND (Y), OR (O) y NOT (NO), que permiten evaluar múltiples situaciones al mismo tiempo. Su correcta comprensión evita errores en la ejecución de condicionales y bucles.
Características:
  • Combinan y comparan condiciones.
  • Principales: AND, OR, NOT.
  • Permiten decisiones complejas.
  • Usados en validaciones y control de flujo.
 Ejemplo: ¿Es azul Y es grande? Solo se cumple si ambas condiciones son verdaderas. Condicionales Son estructuras que permiten que el programa tome decisiones según si una condición se cumple o no. Facilitan el control del flujo del algoritmo, asegurando que se ejecuten las acciones correctas en cada situación. Se usan para validar datos, controlar bucles y ejecutar diferentes procesos según las circunstancias.
Características:
  • Evalúan condiciones y determinan acciones.
  • Clave para control de flujo.
  • Se combinan con operadores lógicos.
  • Usadas en validaciones y decisiones.
 Ejemplo: Si llueve, usar paraguas; si hace sol, usar gafas de sol. If / Else Estructura de control que permite ejecutar un bloque de código si una condición es verdadera (IF) y otro bloque si es falsa (ELSE). Facilita decisiones binarias dentro de un programa y evita la repetición de código. Su correcto uso garantiza que el flujo del algoritmo sea claro, organizado y predecible.
Características:
  • Ejecuta bloques según condición verdadera o falsa.
  • Evita duplicación de código.
  • Claves para decisiones binarias.
  • Base de la programación condicional.
 Ejemplo: IF (edad >= 18) { "Pasa"; } ELSE { "Espera"; } Pseudocódigo Es una forma de escribir la lógica de un algoritmo usando un lenguaje natural o simplificado. Permite planificar soluciones, identificar errores y organizar ideas antes de programar. Facilita la comunicación entre programadores y la comprensión de los procesos para personas sin conocimientos técnicos.
Características:
  • Lenguaje natural o simplificado.
  • Planifica y organiza la lógica.
  • Facilita comunicación y documentación.
  • Permite detectar errores antes de programar.
 Ejemplo: Escribir "Hola", sumar dos números y mostrar el resultado. Diagramas de flujo Representación gráfica de los pasos de un algoritmo mediante símbolos estandarizados: rectángulos para procesos, rombos para decisiones y flechas para indicar flujo. Facilita la comprensión visual de la lógica y permite detectar errores antes de codificar. Es útil en enseñanza y planificación de proyectos complejos.
Características:
  • Representación gráfica de algoritmos.
  • Usa símbolos estandarizados (óvalo, rectángulo, rombo).
  • Flechas indican la secuencia de pasos.
  • Facilita análisis, documentación y mejora de procesos.
 Ejemplo: Rectángulo para "sumar números", rombo para "¿es mayor que 10?", flechas conectando pasos.

Segundo Trimestre – Control y lógica

Bucles Los bucles son estructuras que permiten repetir un conjunto de instrucciones automáticamente hasta que se cumpla una condición. Facilitan la automatización de tareas repetitivas, evitando la escritura duplicada de código. Pueden controlarse mediante condiciones o un número definido de repeticiones y son fundamentales para procesos iterativos y eficientes en programación. Comprender su funcionamiento es clave para desarrollar algoritmos más complejos y dinámicos.
Características:
  • Repetición automática de instrucciones.
  • Controlados por condiciones o número de repeticiones.
  • Permiten automatizar tareas repetitivas.
  • Fundamentales para programación eficiente.
 Ejemplo: Un ventilador que gira mientras esté encendido, repitiendo constantemente la acción. For El bucle "for" se utiliza cuando se conoce el número exacto de repeticiones que se deben realizar. Permite iterar sobre listas, rangos de números o elementos de manera controlada, siendo especialmente útil en cálculos, generación de secuencias y procesamiento de datos. Su sintaxis clara facilita la lectura y comprensión del código, haciendo que la programación sea más estructurada y predecible.
Características:
  • Itera un número definido de veces.
  • Ideal para listas y rangos de datos.
  • Sintaxis clara y estructurada.
  • Usado en cálculos y secuencias controladas.
 Ejemplo: Repetir 10 veces una canción o imprimir números del 1 al 10. While El bucle "while" ejecuta instrucciones mientras una condición específica sea verdadera. Es útil cuando no se conoce de antemano cuántas veces se debe repetir la acción, permitiendo crear programas más flexibles y dinámicos. Si no se define correctamente la condición, puede generar bucles infinitos, por lo que es importante un uso cuidadoso.
Características:
  • Se ejecuta mientras la condición sea verdadera.
  • No requiere conocer el número de repeticiones.
  • Permite procesos flexibles y dinámicos.
  • Debe manejarse cuidadosamente para evitar bucles infinitos.
 Ejemplo: Mientras haya comida en el plato, seguir comiendo hasta que se termine. Do While El bucle "do while" ejecuta primero las instrucciones y luego evalúa la condición para decidir si repetir. Esto garantiza que el bloque de código se ejecute al menos una vez, siendo ideal para situaciones donde se necesita realizar una acción inicial antes de evaluar condiciones. Se usa para validar entradas de usuario o realizar comprobaciones previas a un proceso repetitivo.
Características:
  • Ejecuta el bloque de código al menos una vez.
  • Evalúa la condición después de la primera ejecución.
  • Útil para validaciones iniciales.
  • Permite repetir procesos según condiciones.
 Ejemplo: Probar un bocado de comida y luego decidir si se desea comer más. Contadores Un contador es una variable que aumenta o disminuye de manera controlada, generalmente en incrementos de uno. Se utiliza para llevar registro del número de iteraciones o eventos ocurridos durante la ejecución de un programa. Es fundamental en bucles y en la medición de procesos repetitivos, permitiendo evaluar progresos y controlar ciclos con precisión.
Características:
  • Lleva registro de iteraciones o eventos.
  • Incrementa o decrementa valores controladamente.
  • Fundamental en bucles y conteos.
  • Permite controlar procesos con precisión.
 Ejemplo: c = c + 1, usado para contar cuántos usuarios han iniciado sesión. Acumuladores Un acumulador es una variable que suma o acumula valores a lo largo del tiempo. Se utiliza para calcular totales, promedios o resultados agregados dentro de un programa, ayudando a procesar datos de manera eficiente. Es fundamental en operaciones financieras, estadísticas o cualquier escenario que requiera sumar progresivamente datos.
Características:
  • Suma o acumula valores durante la ejecución.
  • Útil para totales, promedios o agregados.
  • Fundamental en estadísticas y finanzas.
  • Permite mantener un historial de cálculos.
 Ejemplo: total = total + precio, para calcular el costo total de un carrito de compras. Funciones Una función es un bloque de código reutilizable que realiza una tarea específica y puede recibir datos de entrada. Permite organizar el código, reducir la repetición y mejorar la legibilidad. Las funciones pueden devolver resultados, facilitando la modularización y reutilización en diferentes partes del programa.
Características:
  • Bloques de código reutilizables.
  • Realizan tareas específicas.
  • Pueden recibir parámetros y devolver resultados.
  • Mejoran organización y legibilidad del código.
 Ejemplo: function Saludar() { alert("Hola"); }. Parámetros Los parámetros son valores que se envían a una función para que realice su tarea con datos específicos. Permiten personalizar la ejecución de la función según las necesidades de cada situación y facilitan la reutilización del código.
Características:
  • Valores enviados a funciones.
  • Permiten personalizar la ejecución.
  • Facilitan reutilización de funciones.
  • Mantienen el código limpio y flexible.
 Ejemplo: En Calcular(a, b), 'a' y 'b' son parámetros. Retorno El retorno es el valor que una función devuelve después de realizar su operación. Permite que los resultados de la función se utilicen en otras partes del programa y es esencial para cálculos y consultas de datos.
Características:
  • Devuelve un valor tras ejecutar la función.
  • Permite usar resultados en otras partes del programa.
  • Clave para cálculos y consultas.
  • Mejora modularidad y claridad del código.
 Ejemplo: return resultado; devuelve el cálculo realizado. Errores comunes Los errores comunes son fallas al escribir código, como errores de sintaxis, tipográficos o lógicos. Detectarlos y corregirlos es vital para el correcto funcionamiento de cualquier programa. Aprender a identificar patrones de errores frecuentes ayuda a programar de manera más eficiente y reduce el tiempo de depuración.
Características:
  • Pueden ser sintaxis, tipográficos o lógicos.
  • Detectarlos es crucial para buen funcionamiento.
  • Identificar patrones de errores frecuentes ayuda a programar mejor.
  • Prevención reduce tiempo de depuración.
 Ejemplo: Olvidar un punto y coma (;) al final de una instrucción. Depuración La depuración es el proceso de localizar y corregir errores en un programa. Implica revisar el código línea por línea, analizar valores de variables y verificar la lógica de las instrucciones. Es un paso fundamental en el desarrollo de software profesional, asegurando que el programa funcione correctamente.
Características:
  • Localiza y corrige errores.
  • Revisión línea por línea y análisis de variables.
  • Fundamental en desarrollo profesional.
  • Herramientas como depuradores facilitan el proceso.
 Ejemplo: Revisar línea por línea para encontrar un cálculo incorrecto en un bucle. Buenas prácticas Las buenas prácticas son normas y hábitos que permiten escribir código claro, mantenible y profesional. Incluyen nombrar variables coherentemente, comentar funciones importantes y estructurar el código de manera lógica. Facilitan la colaboración entre programadores y reducen errores futuros, garantizando software más robusto y fácil de actualizar.
Características:
  • Nombrado coherente de variables y funciones.
  • Comentarios en funciones importantes.
  • Estructura lógica y clara del código.
  • Facilitan mantenimiento y colaboración.
 Ejemplo: Comentar cada función y usar nombres descriptivos.

Tercer Trimestre – Avanzado

Arreglos Los arreglos son estructuras de datos que almacenan múltiples valores bajo un mismo nombre de variable. Permiten acceder a cada elemento mediante un índice, facilitando operaciones como búsqueda, ordenamiento y modificación. Son esenciales para manejar listas de datos de manera eficiente y pueden contener números, texto o incluso otros arreglos (multidimensionales).
Características:
  • Almacenan múltiples valores en una sola variable.
  • Acceso a elementos mediante índices.
  • Permiten operaciones como búsqueda y ordenamiento.
  • Pueden ser unidimensionales o multidimensionales.
 Ejemplo: [1, 2, 3, 4] representa una lista de números que podemos recorrer con un bucle. Índices Los índices indican la posición de un elemento dentro de un arreglo o lista, comenzando normalmente desde 0. Permiten acceder, modificar o eliminar valores específicos de manera directa y son fundamentales para evitar errores como "fuera de rango".
Características:
  • Identifican la posición de un elemento.
  • Generalmente comienzan en 0.
  • Permiten acceso, modificación y eliminación directa.
  • Evitan errores de desbordamiento o fuera de rango.
 Ejemplo: En ["manzana","pera","uva"], "pera" está en el índice 1. Matrices Las matrices son arreglos de dos o más dimensiones, similares a tablas con filas y columnas. Permiten almacenar datos complejos, como tablas de puntajes, imágenes o mapas de celdas. Cada posición se identifica con un índice de fila y columna y son ampliamente usadas en ciencias, ingeniería y desarrollo de videojuegos.
Características:
  • Almacenamiento de datos en filas y columnas.
  • Acceso mediante índices de fila y columna.
  • Permiten manejar información compleja.
  • Usadas en ciencia, ingeniería y videojuegos.
 Ejemplo: [[1,2],[3,4]] representa una matriz 2x2. Objetos Un objeto representa una entidad del mundo real con propiedades y comportamientos. Permite agrupar datos y funciones relacionadas dentro de una misma estructura. Facilita la programación orientada a objetos y mejora la organización del código.
Características:
  • Representa entidades del mundo real.
  • Contiene propiedades y métodos.
  • Facilita la organización del código.
  • Fundamental en programación orientada a objetos.
 Ejemplo: Objeto Persona con propiedades nombre, edad y método saludar(). Clases Las clases son plantillas para crear objetos con propiedades y métodos predefinidos. Permiten generar múltiples instancias sin repetir código y son la base de la programación orientada a objetos, facilitando la reutilización y organización de proyectos grandes.
Características:
  • Plantillas para crear objetos.
  • Incluyen propiedades y métodos predefinidos.
  • Permiten crear múltiples instancias.
  • Facilitan la reutilización y organización del código.
 Ejemplo: Clase 'Perro' para crear 'Fido' y 'Toby', cada uno con sus atributos y comportamientos. Métodos Los métodos son funciones asociadas a un objeto o clase que permiten que los objetos realicen acciones o cálculos basados en sus propiedades. Organizan el código, evitan duplicación de funciones y son esenciales para la encapsulación y control de comportamiento en programación orientada a objetos.
Características:
  • Funciones asociadas a objetos o clases.
  • Permiten ejecutar acciones o cálculos.
  • Organizan el código y evitan duplicación.
  • Claves para encapsulación y control de comportamiento.
 Ejemplo: perro.ladrar() ejecuta la acción de ladrar del objeto. Eventos Los eventos son acciones que ocurren en un programa y que pueden ser detectadas y manejadas. Pueden generarse por interacción del usuario, cambios en datos o tiempo transcurrido y permiten crear aplicaciones interactivas y dinámicas.
Características:
  • Acciones detectables en el programa.
  • Generados por usuario, datos o tiempo.
  • Permiten interacción y dinamismo.
  • Clave en desarrollo web y aplicaciones modernas.
 Ejemplo: onclick ejecuta una función al hacer clic en un botón. DOM El DOM (Document Object Model) es la estructura que representa un documento HTML como un árbol de elementos. Permite acceder y modificar contenido, estilo y estructura de la página mediante código, siendo esencial para crear páginas dinámicas e interactivas.
Características:
  • Estructura jerárquica de elementos HTML.
  • Permite acceso y modificación de contenido y estilo.
  • Esencial para páginas interactivas.
  • Facilita manipulación eficiente del documento.
 Ejemplo: document.getElementById("miDiv").style.backgroundColor = "red". Validaciones Las validaciones verifican que los datos ingresados por el usuario sean correctos y seguros antes de procesarlos. Previenen errores, inconsistencias y vulnerabilidades, asegurando un correcto funcionamiento de la aplicación y una buena experiencia de usuario.
Características:
  • Comprobación de datos ingresados.
  • Prevención de errores y vulnerabilidades.
  • Esencial para seguridad y confiabilidad.
  • Fundamental para experiencia del usuario.
 Ejemplo: Comprobar que el campo correo electrónico no esté vacío y tenga formato correcto. Proyectos Los proyectos integran todos los conocimientos adquiridos durante el curso, aplicando teoría en desarrollos reales. Permiten enfrentar problemas, buscar soluciones y consolidar habilidades de programación avanzada, preparación clave para escenarios profesionales.
Características:
  • Integran conocimientos del curso.
  • Aplican teoría en proyectos reales.
  • Fortalecen habilidades de programación.
  • Preparan para escenarios profesionales.
 Ejemplo: Crear una app de tareas que permita agregar, eliminar y marcar tareas completadas. Lógica compleja La lógica compleja combina estructuras de control, bucles, funciones y objetos para resolver problemas avanzados. Permite optimizar procesos, automatizar decisiones y desarrollar algoritmos sofisticados, base de sistemas inteligentes y programas de alto nivel.
Características:
  • Combina bucles, funciones y objetos.
  • Permite resolución de problemas avanzados.
  • Optimiza procesos y decisiones automáticas.
  • Base de sistemas inteligentes y programas complejos.
 Ejemplo: Crear un motor de búsqueda que filtre resultados según relevancia y preferencias del usuario.
